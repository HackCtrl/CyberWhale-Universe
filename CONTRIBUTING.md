# Contributing to CyberWhale Universe

Thanks for your interest! To keep the project healthy:

- Create feature branches named `feat/<short-desc>` or `fix/<short-desc>`.
- Open pull requests against `main` and provide screenshots or gifs for UI changes.
- Add tests for new logic where applicable.
- Run `npm install` and `npm run dev` before pushing.

Code style
- Follow existing code patterns and Tailwind utility classes.
- Keep components small and reusable.

Review process
- PRs require at least one approver.
- Merge with squash when approved.
