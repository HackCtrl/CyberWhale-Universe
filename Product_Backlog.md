# Product Backlog (MoSCoW) — начальный вариант

Дата: 2026-02-10

Цель: собрать основной набор фич для MVP и приоритизировать по методу MoSCoW.

Must have
- Регистрация/авторизация (email + пароль)
- Личный кабинет: отображение прогресса и активных курсов
- Модель курса/урока в БД (Course, Lesson, UserProgress)
- CTF: создание задач и проверка флагов (минимальный набор)
- Docker + CI для сборки и деплоя

Should have
- Генерация сертификатов (PDF)
- Интеграция с GigaChat для помощи по задачам
- Каталог курсов с фильтрацией

Could have
- Система подсказок и сохранённые подсказки для CTF
- Более продвинутая аналитика пользователей

Won't have (для MVP)
- Многоуровневые роли и платная подписка (в первом релизе)

Примечания:
- Это минимальный список — будет уточняться в задаче `1.1.2` и обновляться по мере обсуждений.
