# Дизайн‑система — MVP платформы CyberWhale Universe

Дата: 24.01.2026

Краткое описание
- Этот документ фиксирует цветовую палитру, типографику и набор базовых UI‑компонентов для MVP.
- Токены стилей находятся в `src/styles/design-tokens.json`.
- Базовые CSS‑реализации смотрите в `src/styles/theme.css` и `src/styles/fonts.css`.

---

## 1. Цветовая палитра
(Основные значения заданы в `src/styles/design-tokens.json` и `src/styles/theme.css`)

- Основной фон: `--background` = #0f1724 (тёмный фон)
- Основной цвет текста (foreground): `--foreground` = #e6e9ef
- Акцент / Primary: `--primary` = #9fef00 (неоново‑зелёный)
- Secondary: `--secondary` = #60a5fa (электрический синий)
- Surface (карточки/панели): `--surface` = rgba(16,24,39,0.6)
- Muted текст: `--mutedForeground` = #94a3b8
- Статусы: `--success` = #34d399, `--warning` = #f59e0b, `--danger` = #ef4444

Рекомендации по использованию
- Primary — для главных CTA, активных состояний и акцентов (использовать также glow для хайлайта).
- Secondary — для вторичных кнопок, ссылок и графиков.
- Surface — карточки, панели и модальные окна (контраст на dark фоне).
- Проверять контраст (WCAG AA) при добавлении новых фонов/текста; в проекте есть скрипт `scripts/check-contrast.*` для проверки токенов.

---

## 2. Типографика
(Подключение шрифтов: `src/styles/fonts.css`)

- Заголовки: `Orbitron` (где требуется футуристический акцент). Размеры заданы в токенах:
  - H1: 48px / 56px
  - H2: 36px / 44px
  - H3: 28px / 36px
  - H4: 20px / 28px
  - H5/H6: 16/14px
- Тело и интерфейс: `Roboto Mono` (моноширинный стиль выбран для атмосферного визуала и читабельности).
  - Body: 16px / 24px
  - Caption: 12px / 16px
- Кодовые блоки: использовать `Roboto Mono` (моноширинная гарнитура), подсветку синтаксиса по вкусу (не включено в MVP).

Рекомендации
- Orbitron для крупных заголовков/брендинга, но избегать его в плотных текстовых блоках — использовать Roboto Mono.
- Всегда указывать fallback‑шрифты: sans‑serif для заголовков, monospace для тела (как в `theme.css`).

---

## 3. Токены и переменные
- Файл токенов: `src/styles/design-tokens.json` содержит цвета, радиусы, тени, типографику, motion и пр.
- CSS-переменные и классы реализованы в `src/styles/theme.css`.
- Touch target: `--touchTarget` = 44px — соблюдать при проектировании кнопок и интерактивных элементов.

---

## 4. Базовые UI‑компоненты (описание и рекомендации)

1) Кнопки (Button)
- Варианты: `primary`, `secondary`, `ghost`, `danger`, `link`.
- Стандарты:
  - Высота: минимум `--touchTarget` (44px).
  - Радиус: `--radius` (8px).
  - Тень: `--shadow-card` для elevated buttons.
  - Фокус: outline с `--ring` и полупрозрачной обводкой.
- Класс CSS: `.cw-btn` (реализовано в `theme.css`).
- Примеры использования: `primary` — главный CTA (on-hero, формы), `secondary` — альтернативные действия.

2) Поля ввода (Input, Textarea, Select)
- Класс: `.cw-input`.
- Стандарты:
  - Фон: `--input-background`; бордер: `--border`.
  - Фокус: border-color `--primary` + glow и box-shadow.
  - Ошибка: border-color `--danger` и aria-invalid.
- Высота и padding: соответствуют touch target и визуальной иерархии.

3) Карточки (Card)
- Класс: `.cw-card`.
- Стандарты:
  - Фон: `--surface`, бордер: `--border`, радиус: `--radius`.
  - Тень: `--shadow-card`; hover — `--shadow-elevated`.
  - Использовать `cw-card` для курс-карточек, карточек в ЛК, виджетов.

4) Навигация / Хедер / Сайдбар
- Хедер: фиксированный (sticky), слева логотип, центр — навигация, справа — действующие кнопки.
- Сайдбар (ЛК): вертикальная навигация, фон `--sidebar`, элементы с `--sidebar-foreground`.
- Активный пункт навигации — цвет `--primary` и indicator (left border или glow).

5) Прогресс‑бар
- Класс: `.cw-progress` и `.cw-progress__indicator`.
- Высота: `--progress-height`; bg: `--progress-bg`; индикатор — градиент от `--primary` к `--secondary`.

6) Мелкие компоненты
- Теги/Pill: маленькие метки уровня/длительности — фон `rgba(255,255,255,0.04)` с границей и небольшим радиусом.
- Spinner: `.cw-spinner` — минимальный индикатор загрузки.
- Модальные окна: фон `--popover`, тень `--shadow-elevated`.

---

## 5. Accessibility (кратко)
- Все интерактивные элементы имеют `aria-label`/`aria-pressed`/`aria-invalid` где требуется.
- Минимальный размер touch target — 44px.
- Контраст текста/фонов — проверять по WCAG AA; для токенов есть `scripts/check-contrast.*`.
- Клавиатурная навигация и видимая фокусная индикация обязательны.
