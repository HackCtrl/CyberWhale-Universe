# Вайрфреймы — MVP образовательной платформы

Дата: 24.01.2026

Фокус: низкодетализированные (lo‑fi) вайрфреймы ключевых экранов: Главная, Личный кабинет, Страница курсов, Страница CTF.

Общие правила раскладки
- Хедер: логотип слева, глобальная навигация (Курсы, CTF, Сообщество), действия справа (Войти / Профиль).
- Боковая навигация (в ЛК): вертикальная, слева — навигация по курсам/настройкам/сертификатам.
- Контентная зона: основная колонка с карточками/контентом, вспомогательная правая колонка для виджетов/рекомендаций (опционально).
- Футер: ссылки, контакты и копирайт.

Примечание после ревью:
- Хедер должен быть фиксированным при скролле (sticky) и показывать активную страницу.
- Все CTA обязаны иметь `aria-label` и минимальную высоту 44px для улучшения доступности.


## Экран 1 — Главная страница

Структура (вертикально):
- Хедер
  - Левый блок: логотип
  - Центр: ссылки "Курсы", "CTF", "О нас"
  - Право: кнопки "Войти" / "Регистрация"
- Герой-блок (широкая панель)
  - Заголовок: ценностное предложение
  - Подзаголовок: краткое описание
  - CTA: "Каталог курсов" и "Начать CTF"
    - Главный CTA должен быть визуально приоритетным: контрастный цвет, увеличенный padding и четкая `aria-label`.
  - Иллюстрация/фон
- Секция "Популярные курсы" (сетка карточек 3/4 в ряд)
  - каждая карточка: миниатюра, название, метки (бесплатный/платный), кнопка "Подробнее" / "Записаться"
- Секция преимуществ (иконки + тексты)
- Секция отзывов (карточки)
- Футер

Условная разметка (текст):
HEADER: [Logo] [Курсы | CTF | Сообщество] [Войти | Регистрация]
HERO: [H1] [P] [CTA1][CTA2] [Image]
POPULAR: [Card][Card][Card]
FEATURES: [Icon+Text]
REVIEWS: [Quote]
FOOTER


## Экран 2 — Личный кабинет (Dashboard)

Цель: быстрый доступ к текущим курсам и прогрессу.

Структура (двухколоночная):
- Хедер (как на сайте)
- Основная область (левая широкая колонка)
  - Приветствие и общий прогресс пользователя
  - Блок "Мои курсы" — список карточек курса (вертикально)
    - Для каждой карточки: прогресс бар, кнопка "Продолжить"
  - Последняя активность (логи, уведомления)
- Боковая колонка (право)
  - Быстрые ссылки: Настройки, Сертификаты, История оплат
  - Рекомендации курсов
  - В карточках "Мои курсы" добавить мини‑прогрессбар и кнопку "Ресурсы" (прямой доступ к материалам).
- Футер

Условная разметка:
HEADER
DASH_TOP: [Приветствие][Общий прогресс]
MAIN_LEFT: [Мои курсы: Card x N]
SIDEBAR: [Настройки] [Сертификаты] [Рекомендации]
FOOTER


## Экран 3 — Страница каталога / Карточка курса

Два состояния: (A) Каталог, (B) Карточка курса

(A) Каталог
- Хедер
- Фильтры и поиск (горизонтально над списком)
  - Категории, Уровень, Цена, Сортировка
- Сетка карточек: миниатюра, заголовок, краткое описание, метки, кнопка "Записаться"
- Пагинация внизу

(B) Карточка курса (детальная)
- Верх: заголовок курса, рейтинг, кнопка "Записаться"/"Купить"
- Два столбца:
  - Левый (широкий): описание, программа (список модулей), примеры уроков, отзывы
  - Правый (узкий): карточка с ключевой информацией (преподаватель, длительность, цена, расписание, CTA)
  - Правый (узкий): карточка с ключевой информацией (преподаватель, длительность, цена, расписание, CTA)
    - Отобразить теги: уровень, длительность в виде pill‑меток.
 - Внизу: похожие курсы, FAQ

Разметка (карточка):
HEADER
COURSE_HEADER: [Название][Рейтинг][CTA]
CONTENT: [Описание | Программа | Уроки]
ASIDE: [Инфоблок | Купить]
RELATED
FOOTER


## Экран 4 — Страница CTF

Цель: доступ к задачам CTF, правилам и участию.

Структура:
- Хедер
- Верх: краткий блок с названием события, датами, кнопкой "Участвовать" (если открыто)
 - Верх: краткий блок с названием события, датами, кнопкой "Участвовать" (если открыто)
  - В верхнем блоке добавить таймер события и кнопку "Правила" рядом с CTA.
 - Секция с правилами/описанием (скрываемая/разворачиваемая)
- Список задач (таблица)
  - Строка задачи: название, категория, стоимость/баллы, статус (открыта/закрыта), кнопка "Открыть"
- Правая колонка: таблица лидеров (топ‑10), таймер события
- Нижняя часть: лог отправки флагов, история попыток (по пользователю)

Разметка:
HEADER
CTF_HEADER: [Название][Даты][CTA Участвовать]
CTF_BODY: [Правила][Tasks Table][Leaderboard][Timer]
FOOTER


## Accessibility & Responsive notes
- На мобильных экранах: переход к одно-колоночной компоновке; хедер сворачивается в гамбургер; боковая навигация — скрытая панель.
- Все CTAs должны быть доступного размера и контрастны.

Дополнительно после ревью:
- Убедиться, что все интерактивные элементы доступны по клавиатуре и имеют `aria-label`.
- CTA и элементы управления должны иметь достаточный контраст с фоном (WCAG AA).


## Acceptance criteria (кратко)
- Для каждого экрана описаны ключевые блоки и CTA.
- Доступна текстовая разметка (копируемая) для передачи дизайнерам/разработчикам.
- Есть инструкции по адаптиву и заметки по доступности.
