# QA чек-лист и задачи для проверки User Flows

Дата: 24.01.2026

## Acceptance criteria (кратко)
- Регистрация: позитивный + негативный сценарий; редирект в ЛК.
- Вход: успешная аутентификация; восстановление пароля.
- Карточка курса: отображение всех полей и кнопки записи.
- Запись: бесплатная — мгновенно; платная — корректная обработка оплаты.
- ЛК: отображение курсов, прогресс, переходы на уроки.
- CTF: доступность, участие, сохранение результатов.

## QA сценарии (шаги)
1. Регистрация — позитивный сценарий
   - Открыть главную → Нажать Регистрация → Ввести валидные данные → Убедиться в редиректе в ЛК.
2. Регистрация — негативный сценарий
   - Попробовать зарегистрировать уже существующий email → Увидеть ошибку.
3. Вход — неверный пароль
   - Попробовать войти с неверным паролем → Увидеть сообщение об ошибке и ссылку восстановления.
4. Просмотр каталога
   - Проверить фильтры и поиск на нескольких курсах.
5. Запись на бесплатный курс
   - Записаться, проверить появление курса в "Мои курсы".
6. Запись на платный курс (эмуляция оплаты)
   - Пройти оплату в тестовом режиме, подтвердить запись.
7. Личный кабинет — прогресс
   - Пройти часть урока/теста и проверить, что прогресс сохранился.
8. CTF — участие
   - Перейти в `/ctf`, участвовать в задаче, проверить сохранение результата и отображение в профиле.

## Рекомендации для QA
- Использовать тестовых пользователей: `user_new`, `user_student`, `admin_test`.
- Протестировать на мобильном и десктопе.

- QA: Регистрация — негативные сценарии
- QA: Вход и восстановление пароля

- QA: Запись на бесплатный курс
- QA: Запись на платный курс (тестовая оплата)
