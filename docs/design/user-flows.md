# User Flows — CyberWhale MVP

Цель: описать ключевые пути пользователя для MVP.

1) Гость → Регистрация → Вход → Просмотр каталога курсов
- Гость переходит на страницу /courses
- Нажимает Register → заполняет email/password → получает подтверждение
- После регистрации выполняет Login → получает JWT → вставляет в клиент (или хранится в cookies)
- Открывает курс → просматривает уроки → отмечает прогресс (планируется)

2) Пользователь → Прохождение курса
- Пользователь заходит в ЛК → видит активные курсы и прогресс
- Выбирает курс → открывает урок → нажимает "Завершить урок" → прогресс курса обновлён

3) Пользователь → CTF
- Пользователь заходит в раздел CTF → видит список задач → открывает задачу → вводит флаг → система проверяет → начисляет очки

Примечание: текущая реализация фич по auth и CRUD курсов покрывает часть первого потока; прогресс и флаг-проверка будут реализованы в следующих задачах.
