# QA чек‑лист для вайрфреймов

Дата: 24.01.2026

Цели проверки
- Соответствие структуры экрана описанию вайрфрейма.
- Наличие основных блоков и CTA.
- Адаптивная логика: переход на мобильную одно-колоночную компоновку.

Проверочные шаги
1. Главная
  - Проверить наличие хедера с логотипом и навигацией.
  - Проверить наличие герой-блока с CTA.
  - Убедиться, что сетка карточек представлена.
2. Личный кабинет
  - Проверить приветственный блок и общий прогресс.
  - Проверить список "Мои курсы" и кнопки "Продолжить".
  - Проверить боковую панель с быстрыми ссылками.
3. Каталог / Карточка курса
  - Проверить фильтры и поиск над сеткой.
  - Проверить подробную карточку курса (левый контент + правый сайдбар).
4. CTF
  - Проверить блок с датами и CTA "Участвовать".
  - Проверить таблицу задач и таблицу лидеров.
5. Адаптивность
  - На мобильном: хедер сворачивается, контент становится одно-колоночным.

- Дизайнер/Dev: Создать адаптивные шаблоны на основе вайрфреймов
- QA: Проверка соответствия и адаптива
